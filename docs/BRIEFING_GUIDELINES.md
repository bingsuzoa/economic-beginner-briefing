# 경제 브리핑 작성 가이드라인

## 목적

이 프로젝트는 단순한 뉴스 요약 서비스가 아니다.

목표는 경제 초보가

&gt; "오늘 내 돈에 영향을 주는 일이 있었는가?"

를 3~5분 안에 이해하도록 만드는 것이다.

경제신문을 읽은 것 같은 느낌이 아니라

"오늘 나에게 필요한 경제 정보만 빠르게 확인했다."

라는 경험을 제공하는 것이 목적이다.

---

# 핵심 원칙

브리핑은 뉴스를 요약하는 것이 아니라

사용자의 관점에서 의미를 설명해야 한다.

항상 아래 질문을 먼저 생각한다.

- 왜 중요한가?

- 누구에게 중요한가?

- 나랑 무슨 상관인가?

기사에 없는 내용을 추측해서는 안 된다.

---

# 브리핑 구조

기사 하나는 아래 순서로 작성한다.

```

제목

한 줄 결론

왜 중요한가

누가 꼭 읽어야 하나?

무슨 일이 있었나

왜 이런 일이 발생했나

우리에게 어떤 의미가 있나

경제용어

출처

```

---

# 제거할 항목

다음 항목은 기본적으로 출력하지 않는다.

## 앞으로 지켜볼 내용

삭제한다.

이유

대부분의 경우

- 추가 발표 가능성

- 정부 대응 예정

- 시장 반응

처럼 누구나 예상 가능한 내용이 반복된다.

사용자에게 실질적인 도움이 되지 않는다.

정말 중요한 일정(기준금리 발표일, 청약 일정 등)이 있는 경우에만 출력한다.

---

## 지금 확인할 것

기본적으로 출력하지 않는다.

다음과 같은 문장은 정보 가치가 낮다.

- 정부 공지 확인

- 계약서 확인

- 투자에 유의

대부분 모든 기사에 동일하게 사용할 수 있기 때문이다.

실제 행동이 필요한 경우에만 한 줄 정도 출력한다.

예시

✔ 사전청약 당첨자라면 국토부 공지를 확인하세요.

---

# 새롭게 추가할 항목

## 누가 꼭 읽어야 하나?

반드시 추가한다.

예시

```

👤 이 뉴스를 꼭 읽어야 하는 사람

✅ 신혼부부

✅ 사전청약 당첨자

✅ 공공분양 준비자

```

또는

```

👤 이 뉴스를 꼭 읽어야 하는 사람

✅ 미국주식 투자자

✅ ETF 투자자

```

반대로

영향이 거의 없는 대상도 표시할 수 있다.

```

❌ 크게 신경 쓰지 않아도 되는 사람

• 이미 일반 아파트를 계약한 사람

• 전세 거주자

```

---

# 왜 중요한가

이 항목은 기사 내용을 반복하지 않는다.

좋지 않은 예

```

주택 구매를 고려하는 신혼부부에게 중요한 사전청약 관련 이슈입니다.

```

좋은 예

```

사전청약 당시 약속했던 대출 조건이 변경될 경우

실제 자금 계획이 달라질 수 있기 때문입니다.

```

항상

"왜 중요한가"

를 설명해야 한다.

---

# 우리에게 어떤 의미가 있나

가장 중요한 항목이다.

기사 내용을 다시 설명하지 않는다.

사용자가

"그래서 나는 어떻게 이해하면 되는가?"

를 알려준다.

예시

```

공공분양을 준비하는 사람은

앞으로도 정부의 공급 조건이 변경될 가능성을 고려해야 합니다.

이미 일반 아파트를 계약한 사람은

이번 뉴스의 직접적인 영향은 거의 없습니다.

```

---

# 경제용어

경제 초보가 이해하기 어렵다고 판단되는 용어만 설명한다.

설명은

1~2문장 이내로 작성한다.

너무 쉬운 단어는 설명하지 않는다.

예시

좋음

- DSR

- LTV

- ETF

- ISA

- 리츠

불필요

- 집

- 아파트

- 은행

---

# 영향이 없는 뉴스

영향이 없다면

억지로 의미를 만들지 않는다.

예시

```

일반 가계나 개인 재테크에는

직접적인 영향이 거의 없는 뉴스입니다.

특정 기업 간 계약 또는 소송에 가까운 내용입니다.

```

---

# 금지 사항

절대 하지 않는다.

❌ 기사에 없는 미래 예측

❌ 기사에 없는 투자 조언

❌ 기사에 없는 부동산 전망

❌ 기사에 없는 금리 전망

❌ 기사에 없는 행동 유도

예시

```

공사비 소송

↓

집값 상승 가능성

↓

신혼부부는 주택 구매를 신중히 해야 합니다.

```

이런 분석은 생성하지 않는다.

---

# 브리핑 품질 기준

좋은 브리핑은

뉴스를 많이 설명하는 것이 아니라

사용자가

"그래서 나랑 무슨 상관인데?"

를 이해하게 만든다.

다음을 항상 만족해야 한다.

- 기사 내용 반복 금지

- 쉬운 문장 사용

- 불필요한 섹션 제거

- 행동이 필요한 사람만 행동 안내

- 기사에 없는 내용 추론 금지

- 경제 초보 기준으로 작성
